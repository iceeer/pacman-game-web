# 吃豆人游戏设计文档

**日期**: 2026-04-09
**状态**: 已批准

---

## 1. 概述

用纯 HTML + CSS + JavaScript 实现经典吃豆人游戏，单文件交付，浏览器双击即玩。

### 核心功能
- 经典迷宫布局
- 吃豆人 + 4个差异化鬼魂
- 豆子 + 能量球系统
- 计分 + 生命 + 游戏状态管理
- 经典像素风视觉

---

## 2. 技术架构

### 渲染
- Canvas 2D API
- `requestAnimationFrame` 驱动，60fps 固定帧率

### 输入
- 键盘事件：方向键 / WASD
- 空格暂停，Enter 开始/重启

### 游戏状态机
`READY` → `PLAYING` → `PAUSED` → `GAME_OVER` → `WIN`

---

## 3. 数据结构

### 迷宫地图
2D 数组，枚举值：`WALL`, `DOT`, `POWER_PELLET`, `EMPTY`, `GATE`

### 吃豆人
| 属性 | 说明 |
|------|------|
| x, y | 网格坐标 |
| direction | 当前移动方向 |
| nextDirection | 缓冲的下一个转向 |
| lives | 剩余生命（默认3） |
| mouthAngle | 嘴巴张合动画 |

### 鬼魂（4个）
| 属性 | 说明 |
|------|------|
| x, y | 网格坐标 |
| direction | 当前方向 |
| state | SCATTER / CHASE / FRIGHTENED / EATEN |
| color | 各自颜色区分 |

### 全局状态
| 属性 | 说明 |
|------|------|
| score | 当前得分 |
| highScore | 历史最高（localStorage） |
| level | 当前关卡 |
| gameState | 游戏状态机 |

## 4. 游戏逻辑

### 吃豆人移动
- 优先尝试 `nextDirection`，非墙则转向
- 否则沿 `direction` 继续
- 遇墙停止
- 左右边界穿隧道

### 鬼魂AI
- 统一曼哈顿距离追踪算法
- 每个路口选择最短距离方向（不回头）
- 模式切换：
  - **CHASE**: 目标=吃豆人位置
  - **SCATTER**: 目标=各自角落
  - **FRIGHTENED**: 随机方向
  - **EATEN**: 目标=鬼魂老家

### 能量球机制
- 吃下后鬼魂进入 FRIGHTENED（约8秒）
- 吃鬼魂得分：200→400→800→1600
- 被吃鬼魂变眼睛回老家复活

### 碰撞检测
- 基于网格的离散碰撞
- 吃豆人吃豆子 → 加分
- 吃豆人碰鬼魂 → 减命/游戏结束
- FRIGHTENED 期间吃鬼魂 → 加分

### 画面布局

- 鬼魂眼睛朝向移动方向

- 能量球闪烁

### 交互
- 方向键/WASD 控制
- 空格暂停
- Enter 开始/重新开始
